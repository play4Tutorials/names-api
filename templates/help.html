This is the help page of Name API - A demo api made using Python and Flask!


Introduction


Names API has a back-end ofcourse but also a front-end. Names API is hosted and deployed on and using Heroku Platform. The front-end exist at "/" while the backend has 2 urls - "/names/id={id}&name={name}&age={age}&phone={phone}&email={email}" and "/name/{id}". One url is for POST-ing the data while the other is for GET-ing the data. This app uses a SQLite3 Database


The Front-End


The Front-End of this app can be accessed at "/". The front-end is very simple. It uses basic HTML and CSS and no JavaScript. Just some Bootstrap for the Navbar. It consists of a form and a table. You can use the form to POST a new entry to the database. The table contains all the POST-ed data, that can be accessed using a GET request to the back-end API. Next-up is the back-end.


The Back-End


The back-end as mentioned earlier uses Python and Flask deployed on Heroku. It uses a database to store all data from POST requests to the API. It's quite simple. The database is a SQLite3 database using flask_sqlalchemy python package. The API is the most interesting which is up next!


The API


Alright so, the API! The API consists of 2 URLs - 1 for POST and 1 for GET requests.
The POST Request URL - https://my-names-api.herokuapp.com/names/id={id}&name={name}&age={age}&phone={phone}&email={email}
The GET Request URL - https://my-names-api.herokuapp.com/name/{id}
Remeber the id you gave your POST request, because you can access it using only the id! The API has a system that does not allow duplicate id(s). It also has a system that Returns a error if you try to reques an id which doesn't exist! And that's all there is to this Website!
